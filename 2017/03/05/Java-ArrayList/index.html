ArrayList类

发表于 2017-03-05   |   分类于 Java ， JDK   |     |   阅读次数

可调整大小的数组的List接口的一种实现。允许元素为空。有内部调整数组存储大小的方法。大体上与Vector相似，但不同步。

ArrayList<E> 接口

RandomAccess接口: 空接口，标识作用。给可以提供随机访问的List实现去标识一下，这样使用这个List的程序在遍历这种类型的List的时候可以有更高效率。仅此而已。所以，我们在遍历List之前，可以用 if(list instanceof RamdomAccess) 来标识一下，选择用哪种遍历方式。就这点小技巧，完。见于java 集合框架的接口之一：RandomAccess

size, isEmpty, get, set, iterator和listIterator方法在常数时间内完成。add可以在分期摊还的常数时间(amortized constant time)内完成。其他操作一般在限行时间完成。且比LinkedList的常量因子要低。
如果有很多线程并发地共享一个ArrayList实例，并且至少一个线程进行修改表结构(任何添加删除扩容都是)的操作，那么需要外部设置并发操作的。可以用collections.synchronizedlist方法包装使用，这个操作最好在创建对象时。

由方法iterator和listIterator返回的迭代器是支持快速失败的，在迭代器创建后，列表结构的修改(除非是迭代器自带的remove或add方法)都会抛出ConcurrentModificationException。在并发操作时迭代器失败得迅速而干净，而不是在一个不确定的时间进行任意的、不确定的行为。但我们不能依赖这种行为来写代码(它不能强有力地保证没有并发操作)，只能用于发现bug。

方法

初始化方法

这里需注意，ArrayList类用数组elementData存储元素，该elementData.length与ArrayList类的私有成员变量size并不一样。前者有助于减少多次扩容带来的影响，后者为实际list的大小。

  • public ArrayList(int initialCapacity): 创建一个initialCapacity大小的空list。若initialCapacity为0，则list是{}；若initialCapacity小于0，则抛出异常IllegalArgumentException。
  • public ArrayList(): 创建一个初始大小为10的空list。[其实执行之后list还是{}，但由于命名一类的方式，后续操作直接处理了初始大小]
  • public ArrayList(Collection<? extends E> c): 创建一个包含c中所有元素的list，顺序与c中迭代器遍历顺序一致。[toArray()赋予elementData]

List接口的实现

  • public int size(): 返回size。
  • public boolean isEmpty(): 返回 size == 0
  • public boolean contains(Object o): 调用了indexOf()进行判断
  • public int indexOf(Object o): 分了两种情况讨论. o是否为null。[近似于o==null ? get(i)==null : o.equals(get(i)))]
  • public int lastIndexOf(Object o): 方法与indexOf()类似。
  • public Object[] toArray(): 直接调用了Arrays.copyOf(elementData, size)
  • public T[] toArray(T[] a): 设置了数组有效数字后一位为null, 来标明数组长度, 但如果数组本来就接受null元素, 就没办法了。
  • public E get(int index): 读取了数组里索引为index的元素，并强转成所需元素类型。
  • public E set(int index, E element): 设置index索引的元素为element，并返回旧值。
  • public boolean add(E e): 先检查数组长度是否足够（每次内部检查数组长度一定会触发modCount++），再添加。
  • public void add(int index, E element): 添加元素，此时index需要满足小于等于size()并且不为负。
  • public E remove(int index): 删除元素，并返回删除的元素。其中采用了 elementData[–size] = null; 来清空末尾元素。[不知道为什么没有直接调用个fastRemove()函数来操作，反而重复编写了。]
  • public boolean remove(Object o): 删除指定元素第一个出现的地方，返回是否存在该元素。若存在则执行的相当于(o==null ? get(i)==null : o.equals(get(i)))
  • public void clear(): elementData全部置为空，size为0。[这里总会提到elementData中某项置为空时，会触发GC，因为没有指向对象的引用了~]，这里数组大小并没有改变。
  • public boolean addAll(Collection<? extends E> c): 先转换为数组，检查大小，再System.arraycopy()，返回c转换为数组后的长度是否为0:)
  • public boolean addAll(int index, Collection<? extends E> c): 与上者近似，多加了一个数组移动。
  • public boolean removeAll(Collection<?> c): 保留本集合中未在集合c中出现的所有元素。mountCount变化值等于集合数量减少的值。
  • public boolean retainAll(Collection<?> c): 保留本集合中也在集合c中出现的所有元素。mountCount变化值等于集合数量减少的值。
  • public ListIterator listIterator(int index): 返回从index开始的ListIterator(其实是ListItr实例)。
  • public ListIterator listIterator(): 返回从list起始位置开始的ListIterator。
  • public Iterator iterator(): 返回从list起始位置开始的Iterator(内部类Itr实例，该内部类实现了Iterator接口)。

  • List subList(int fromIndex, int toIndex): 返回从fromIndex(包含)到toIndex(不包含)的子List，如果 fromIndex == toIndex 那么返回空List（不是null）。检查参数类型，再新建一个SubList。这里的SubList又是一个内部类。

  • public void sort(Comparator<? super E> c): 对本集合的数组排序，排序前和排序后的mountCount进行对比，避免在操作时数组改变。Java 1.8 特性

  • public void forEach(Consumer<? super E> action): Java 1.8 特性
  • public void replaceAll(UnaryOperator operator): Java 1.8 特性
  • public Spliterator spliterator(): Java 1.8特性
  • public boolean removeIf(Predicate<? super E> filter) : Java 1.8 特性

继承的方法的实现

  • public Object clone(): 返回ArrayList的浅拷贝(shallow copy)，列表元素并没有拷贝。
  • protected void removeRange(int fromIndex, int toIndex): 覆盖AbstractList的方法。SubList和AbstractList调用较多，移动、清空、改变size大小。

Serialization

  • private void writeObject(java.io.ObjectOutputStream s)
  • private void readObject(java.io.ObjectInputStream s)
    用于根据输入输出新建一个实例。

其他的特有方法

  • public void trimToSize(): 将ArrayList实例大小缩减成现在list的大小(elementData.length此时与size一致)。可以用于优化ArrayList实例的存储大小。[用Arrays.copyOf()]
  • public void ensureCapacity(int minCapacity): 确保容量大于minCapacity。（如果空list，这时候容量往往还是大于10的）。用户指定ArrayList实例的容量，可以有效的降低时间成本。

modCount

会触发modCount变化的方法:

trimToSize()，add(), addAll()，ensureCapacity()在需要扩容的时候，remove()，clear()；removeAll()，retainAll()在删除了些元素的时候（modCount此时变化的次数等于removeAll()与retainAll()中改变元素的次数，和addAll()不同可能是由于addAll()调用System.arraycopy()封装了一次移动操作，而这两个是函数内部直接循环改变的。）

不包括set()噢。

扩容函数

ArrayList类无论是外部调用ensureCapacity()函数进行扩容，还是内部因为add()没有空位而进行扩容，都会执行到以下函数:

新容量扩大到原容量的1.5倍，右移一位相关于原数值除以2。如果不够再加到minCapacity, 这样也保证了至少新容量是旧容量的1.5倍, 防止扩容次数太多 :)

MAX_ARRAY_SIZE定义为Integer.MAX_VALUE-8; 即内部数组大小的最大值, 保留8个给一些VM用作数组头部以存储信息。但其实MAX_ARRAY_SIZE的限制也防不住minCapacity啊…(至于hugeCapacity()中选择返回Integer.MAX_VALUE而不是返回minCapacity，是因为就相差8个大小还不如容量最大化呢吧)

应用可以在进行大量添加操作前先进行扩容，会减少重新分配的次数。

The details of the growth policy are not specified beyond the fact that adding an element has constant amortized time cost.

batchRemove(Collection<?> c, boolean complement)

removeAll()调用了该方法，complement参数为false；retainAll()调用了该方法，complement参数为true。很有意思的实现。

ListItr内部类

An optimized version of AbstractList.ListItr （应该先看AbstractList么…然而并没有）

Itr内部类比较容易，实现了Iterator接口的方法。用cursor表示下一个操作数，lastRet用以删除当前操作数(-1代表当前没有操作数，也限定了remove()方法只能删除一个元素)，expectedModCount用以检查数组长度与遍历前是否一致(只有在cursor改变时才需要检查，有种写锁的感觉呢…)。

相对于Iter加了很多方法，重点谈如何实现。

  • ListItr(int index): cursor值设为index。（。。。相当于指针了，反正ArrayList是如此的数组）
  • public boolean hasPrevious(): 是否有先驱。(返回cursor != 0，不写cursor > 0这样，应该是cursor本来也不会被人操作，在负责区域内只要不让它小于0就好)
  • public int nextIndex(): 下一个index是多少。(就是cursor的含义)
  • public int previousIndex(): 上一个index。(这里返回-1的话就说明是现在是第一个元素了，用cursor-1实现的)
  • public E previous(): 返回先驱结点。检查+读取元素，和remove()实现方法略像。读取元素前，拷贝了下elementData的引用。
  • public void set(E e): 将当前遍历的上一个元素设置为e。设置lastRet的值为e。这里不会改变lastRet的! (和remove又不一样)。
  • public void add(E e): 在当前遍历的位置插入元素e。添加后又一次置lastRet为-1。

所有的操作其实都是对外部类进行的，即ArrayList对象进行的。

但形式上分成对建立它的parent进行操作，或者对外部类ArrayList对象进行。包括set()，get()，add()，remove()，removeRange()，addAll()。如果对parent操作呢，就用parentOffset对应index，如果对ArrayList对象操作，就用offset对应index。后者是get()与set()方法，前者是剩下的其他方法。

每次进行了改变modCount操作的，都需要更新modCount。也需要及时更新SubList对象的size。

顺便还涉及了内部匿名类，如ListIterator的定义，iterator()方法也返回了ListIterator内部类。
