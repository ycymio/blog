HashMap类

发表于 2017-03-13   |     |   阅读次数

HashMap基于哈希表实现了Map接口。该操作提供所有的map可选操作，并且key和value值都允许为null（HashMap类与Hashtable类相似，除了在HashMap不同步并允许null元素方面）。

HashMap 类

每个元素的hash值计算是元素原来的hash值的高16位与低16位进行异或操作得到的。

HashMap类不保证map的顺序，尤其不保证遍历顺序一直与插入顺序一致。假设哈希函数使得元素正确投射到相应桶(bucket)的情况下，它的基础操作(get和put)呈现常量时间复杂度。集合的遍历需要HashMap实例的容量(桶的大小)和桶所拥有的key-value的数量之和成正比的时间，所以不要将初始值设置的过高或者负载因子设置的过低，这对遍历上呈现的性能很重要。

一个HashMap实例有两个影响它效果的参数：初始大小和负载因子。哈希表的容量(capacity)指哈希表中桶的数量，初始大小就是一开始创建的容量。负载因子用于衡量哈希表在自动扩容前的满载程度。当一个哈希表的节点数超过了负载因子乘以当前大小，哈希表将重新哈希，内部的数据结构重新构建，使得哈希表有约原先桶的两倍的数量。

一般而言，默认的负载因子(0.75)在时间与空间开销上提供了很好的权衡，更高的值减小了空间开销但增加了查找的开销(反映在绝大多数HashMap类的操作中，包括get和put)。在设置map的初始大小时应考虑预计的entry数量和负载因子，从而减小再哈希的次数。

成员变量

  • transient Node[] table：table，存储hash值。第一次使用时初始化，大小不够则扩容。长度总是2的幂。
  • transient Set> entrySet：保存缓存的entrySet()。
  • transient int size：map的大小。
  • transient int modCount: 被结构修改的次数。
  • int threshold: 当HashMap的size大于threshold时会执行resize操作。最大容量是[1 << 30]。threshold=capacity*loadFactor。
  • final float loadFactor: 哈希表的负载因子。表示散表的装满程度,定义为散列表中节点的数目除以基本区域能容纳的节点数所得的商。

方法

初始化方法

  • public HashMap(int initialCapacity, float loadFactor)：给定最初大小和负载因子，建立空HashMap。
    tableSizeFor(capacity)用于找到大于等于指定容量(capacity)参数的2的幂。
  • public HashMap(int initialCapacity)：给定最初大小，建立空HashMap。(默认DEFAULT_LOAD_FACTOR值为0.75f)
  • public HashMap()：建立空HashMap，默认大小为16，默认负载因子为0.75。
  • public HashMap(Map<? extends K, ? extends V> m)：根据给定的map创建一个新hashmap，负载因子初始值为0.75。调用了putMapEntries()，第二个参数只有在初始化时是false。

Map接口的实现

  • public int size()：返回size。
  • public boolean isEmpty()：返回map是否为空[size == 0]。
  • public V get(Object key)：不存在返回null，否则返回对应的value。查到hash值对应的table项，再一项一项比对key是否引用相同或者值相同。所有的匹配都分为两种：TreeNode和非TreeNode，在此之前，先检查第一项的[因为第一项就可以不进行匹配，冲突少的话比较速度]。
  • public boolean containsKey(Object key)：直接调用get()中调用的getNode()方法，查看返回值是否为null。
  • public V put(K key, V value)：具体调用在后边讲了下。不管之前是否有K-V键值对，都会进行插入/修改操作。返回值是之前key对应的值。
  • public void putAll(Map<? extends K, ? extends V> m)：将新map即m中所有的键值对插入该map中，如果之前存在键值对就覆盖。
  • public V remove(Object key)：如果key存在则返回对应的value，否则返回null。
  • public void clear()：清空了table。
  • public boolean containsValue(Object value)：true：返回键值对是否包含该value。
  • public Set keySet()：返回key的Set型集合。
  • public Collection values()：返回value的集合。
  • public Set> entrySet()：返回key-value键值对。

以下据说是Java 1.8新增方法。

  • public V getOrDefault(Object key, V defaultValue)：返回特定key对应的value，如果没有该key，则返回默认值(defaultValue)。
  • public V putIfAbsent(K key, V value)：如果没有这个key，则添加该K-V键值对，并返回null。否则返回当前对应的key。
  • public boolean remove(Object key, Object value)：只有当key与其对应的value都等于给定值时才删除。
  • public boolean replace(K key, V oldValue, V newValue)：如果当前key对应的value是oldValue的话，就用newValue替代。
  • public V replace(K key, V value)：如果当前key对应对应了某值的话，返回并用新值替代。如果不存在该key，则返回null[不做替代操作]。
  • public V computeIfAbsent(K key, Function<? super K,? extends V> mappingFunction)
  • public V computeIfPresent(K key, BiFunction<? super K,? super V,? extends V>
  • public V compute(K key, BiFunction<? super K,? super V,? extends V> remappingFunction)
  • public V merge(K key, V value, BiFunction<? super V,? super V,? extends V> remappingFunction)
  • public void forEach(BiConsumer<? super K,? super V> action)
  • public void replaceAll(BiFunction<? super K,? super V,? extends V> function)
  • public Object clone()：返回当前map的浅复制。（只复制了引用关系）

一些常量

  • DEFAULT_INITIAL_CAPACITY = 1 << 4：索引的初始容量
  • MAXIMUM_CAPACITY = 1 << 30：索引的最大容量
  • DEFAULT_LOAD_FACTOR = 0.75f：默认负载因子
  • TREEIFY_THRESHOLD = 8：当桶(bucket)上的链表数大于这个值时会转成红黑树
  • UNTREEIFY_THRESHOLD = 6：最小为6是红黑树会转化为链表。（为7时也可以转化）
  • MIN_TREEIFY_CAPACITY = 64：The smallest table capacity for which bins may be treeified. (Otherwise the table is resized if too many nodes in a bin.) Should be at least 4 * TREEIFY_THRESHOLD to avoid conflicts between resizing and treeification thresholds.

内部类

Node

存储的是Node静态内部类. 包括get, set, toString, equals, hashCode。成员变量主要是hash值, key, value和下一个结点。

KeySet、Values和EntrySet

并不是存储了key值/value值/键值对。只是在调用外部类的一些东西。iterator()返回了KeyIterator类型/ValueIterator类型/EntryIterator类型。KeySet[size()，clear()，iterator()，contains(Object o)，remove(Object key)]没有添加，删除是真的删除。Values[size()，clear()，iterator()，contains(Object o)]没有对map的添加删除(但可以clear!!!)。EntrySet[size()，clear()，iterator()，contains(Object o)，remove()]没有添加，删除是真的删除，KeySet类的删除是针对key值，EntrySet类的删除必须key-value值一致才删除。
