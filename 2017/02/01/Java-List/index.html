List接口

发表于 2017-02-01   |   分类于 Java ， JDK   |     |   阅读次数

List是一种有序的集合，用户可以精确控制元素插入的地方、根据index取出元素、查找list中的元素。

List<E> 接口

所属包 java.util
允许重复的元素，允许空元素。（可以实现成不支持重复元素，但我们期望这种情况极少）
最好通过index遍历list中的元素。
List接口有ListIterator迭代器，除却Iterator接口的功能，它还支持插入、替换双向遍历。也可以获得特定位置开始的迭代器。
List接口提供两种查找元素的方法、两种插入与删除的方法。
List接口元素可以是自身，但这种情况下equals和hashCode方法则变得难以控制它的行为。
List接口的某些方法会抛出异常，但这些都是可选的。

方法

对父类方法有扩充含义的方法:

  • Object[] toArray(): 根据一定的顺序将列表复制成数组。toArray(T[] a)也一样。
  • boolean add(E e): 在列表末尾添加元素（可选操作）。
  • boolean remove(Object o): 删除从头到尾第一个与对象o相等的元素。

以下列出了除去父类Collection<E>的方法:

  • E get(int index): 返回列表中指定index位置的元素。
  • E set(int index, E element): 用元素element代替原来列表中index位置的元素（可选操作）。[index需要小于列表大小]
  • void add(int index, E element): 在位置index插入元素element（可选操作），在该位置及其以后的元素向右移动。[index需要小于列表大小]
  • E remove(int index): 删除位置index的元素（可选操作），在该位置及其以后的元素向左移动。返回删除了的元素。// -> TODO: 很好奇List result下result.remove(0)会执行哪一个~
  • int indexOf(Object o): 返回对象o第一次出现的索引index，如果不存在则返回-1。
  • int lastIndexOf(Object o): 返回对象o最后一次出现的索引，如果不存在则返回-1。
  • ListIterator listIterator(): 返回一定顺序下的列表的list iterator(列表迭代器)
  • ListIterator listIterator(int index): 返回一定顺序下的列表的list iterator(列表迭代器)，该迭代器从索引index位置开始（也可以向前遍历的）。
  • List subList(int fromIndex, int toIndex): 返回从fromIndex(包括)到toIndex(不包括)位置的list。如果fromIndex等于toIndex，则返回list为空。由于保留了原有的引用，当引用对象变化时，原列表变化（浅拷贝），但我们之后对原列表进行操作时，返回的列表变得语义不明。
    该方法有利于程序员对子list进行操作，譬如清空列表某范围的元素:

以下是Java 8开始支持的方法:

  • default void sort(Comparator<? super E> c): 根据比较器c提供的方法对列表排序。如果参数为null，并且列表中的所有的元素都已经实现了Comparable接口，那么就按照元素自然顺序进行排序。实现策略: 用toArray()方法转换成数组，再通过数组排序，最后再每个位置set()。这样的话也避免了链表排序时间复杂度为n2log(n)的耗时。如果数组近乎排序完成，它只需要大约n次比较。
  • default void replaceAll(UnaryOperator operator): Replaces each element of this list with the result of applying the operator to that element. If the list’s list-iterator does not support the set operation then an UnsupportedOperationException will be thrown when replacing the first element.
  • default Spliterator spliterator(): Creates a Spliterator over the elements in this list.The Spliterator reports Spliterator.SIZED and Spliterator.ORDERED. Implementations should document the reporting of additional characteristic values.
