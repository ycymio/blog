数据库的事务隔离级别

发表于 2017-02-21   |     |   阅读次数

在看面试的书，刷面试的题遇到的一个问题，想着之前一直拖着没解决，今天就看了下，讲数据库的事务隔离级别。某个一直遗留的错误，至少网上没一个让人理解的解答。但是现今这些都封装好了，太基础的也接触不到。

以下是网上一般的讲解：

  • 一级封锁协议是：事务T在修改数据R之前必须先对其加X锁，直到事务结束才释放。事务结束包括正常结束（COMMIT）和非正常结束（ROLLBACK）。在一级封锁协议中，如果仅仅是读数据不对其进行修改，是不需要加锁的，它不能保证可重复读和不读“脏”数据。二级封锁协议：在一级封锁的基础上事务读数据的时候加s锁，读取之后释放。二级封锁协议可以防止丢失更新，脏读。不能防止不可重复读。三级封锁协议：在一级封锁的基础上事务读数据的时候加s锁，直到事务结束释放。二级封锁协议可以防止丢失更新，脏读，不可重复读。

  • 排他锁【X锁】又称写锁。若事务T对数据对象A加上X锁，事务T可以读A也可以修改A，其他事务不能再对A加任何锁，直到T释放A上的锁。这保证了其他事务在T释放A上的锁之前不能再读取和修改A。

^ 两者都对是不可能的啊。。。

应该是排它锁允许其他事务进行读操作，但不允许加锁。“这保证了其他事务在T释放A上的锁之前不能再读取和修改A。” 这句话并不对。是允许其他事务读取A的。

三级封锁协议还防止了虚读，应该与JDBC中的TRANSACTION_SERIALIZABLE对应。
