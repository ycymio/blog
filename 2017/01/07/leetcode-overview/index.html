LeetCode总览

发表于 2017-01-07   |   分类于 LeetCode   |     |   阅读次数

此为对LeetCode的一些归类和短评。有时候会觉得刷题效果不大，练熟练度还不错，但是不会的题一般换种类型还不会…而且算法太多了，针对LeetCode看看算法也不错。

常规LeetCode

LeetCode 1. Two Sum: 不错的练手题。注意若要求index，此时对数组排序后会打破原有的index。-> 可以开始时就记录index的大小。★★★
LeetCode 2. Add Two Numbers: 很不错的练手题，判断条件的合并，代码的简化。值得Medium难度。★★★
LeetCode 3. Longest Substring Without Repeating Characters: 摆明需要的东西，解决遇到的矛盾，解决不了这条思路就断了，解决得了就继续解决。★★★★★
LeetCode 4. Median of Two Sorted Arrays: 暴力解可以。// TODO: O(log(m+n))的优化。
LeetCode 6. ZigZag Conversion: 暴力解。注意双重循环时，内循环修改外循环的时候，外循环的递增是否正确。注意特殊情况的考虑，极端情况无法进入循环的考虑。
LeetCode 7. Reverse Integer: int型数值反转，重点在溢出的处理（选择更大范围的类型，或者做逆运算判断是否等于原结果)
LeetCode 8. String to Integer (atoi): 实现atoi()方法。忘记考虑有正号的情况了。考虑空格，考虑不符合条件的字符，考虑数值超出范围。& charAt()读取的是char型，如果是数字，要减去’0’。
LeetCode 9. Palindrome Number : 数字反转给了思路，可以半反转的。这时候分为奇数位(如121)和偶数位(如1221)。负数不是回文数字，10的倍数也不是回文。
LeetCode 10. Regular Expression Matching : 一种特别简单的正则表达式。很多细节错误，考虑情况不完全[8次提交错误额]，以后可以写个流程图。★★★★★★ 不是很明白的地方在于，为什么从左到右判断的时间大于从右到左的判断时间。// TODO : DP解。
LeetCode 12. Integer to Roman : 有意思的转换，虽然有点呆
LeetCode 13. Roman to Integer : 这是题没理解就做出来了的题…规则用的是，如果当前的大于以后一位的则减去当前值，否则就加上当前值，再加上最后一位的值就好了…
LeetCode 14. Longest Common Prefix : 暴力解。注意 String.charAt(i)，考虑越界。
LeetCode 15. 3Sum : [微软面试题] 一般解法。注意剪枝。
LeetCode 16. 3Sum Closest: 运用剪枝的基础题。★★★★
LeetCode 17. Letter Combinations of a Phone Number: 递归做会很容易。一般在5ms左右。
LeetCode 19. Remove Nth Node From End of List: 双指针问题。特例写错了。这种题设个新的头结点比较好。★★★
LeetCode 20. Valid Parentheses : 括号匹配，栈
LeetCode 21. Merge Two Sorted Lists: 一不小心就忘记移动某个指针的一题。
LeetCode 22. Generate Parentheses: 用递归（Easy），时间: 9 ms。// TODO: 不够优化，会产生很多的String。（怎么不产生呢？）
LeetCode 23. Merge k Sorted Lists: 最小堆排序。范围控制，建堆时没考虑空元素，取大取小弄反了。★★★★★★
LeetCode 24. Swap Nodes in Pairs: 链表两两反转(Easy)
LeetCode 26. Remove Duplicates from Sorted Array : 删除重复元素(数组有序)(Easy)
LeetCode 27. Remove Element : 删除所有特定值的元素(Easy)
LeetCode 29. Divide Two Integers : Math的题要注意正负数的限制。用动态规划做了。迭代法会比递归快。然而递归转化为迭代的熟练度不够。某种程度也算用了二分查找法。
LeetCode 33. Search in Rotated Sorted Array: 多判断条件学会合并。
LeetCode 34. Search for a Range : 计算有序数组中某值的index范围，二分查找的进化版
LeetCode 38. Count and Say : 字符串基本题。注意起始状态。双重循环时注意数组越界。
LeetCode 35. Search Insert Position : 二分查找，先判断相等速度会快很多。
LeetCode 36. Valid Sudoku : 土方法，不太好。
LeetCode 39. Combination Sum: 组合数和。递归可解。记得还原现场~
LeetCode 41. First Missing Positive: 被人讲懂的一题。★★★★★
LeetCode 42. Trapping Rain Water : 栈
LeetCode 43. Multiply Strings : 记得处理”0000”，当参数就相差一个字母时，尤为注意区分。几个优化: ArrayList -> 数组 60ms到39ms；直接加到result上的 -> 39ms到37ms；一开始就是正序的 -> 37ms到31ms；去除最后检查是否还有进位的问题 -> 31ms到30ms ★★★★★
LeetCode 46. Permutations: 递归(回溯)。
LeetCode 48. Rotate Image: 旋转的题目，可以思考是否可以通过几次更简单旋转获得(Solution的解法)，这里直接旋转了，要考虑奇偶性，分下来验证。
LeetCode 49. Group Anagrams: 暴力解决的时候会觉得很容易。（Map的values()无法强转成List，直接new一个ArrayList）
LeetCode 50. Pow(x, n): 会被各种情况烦死的一题。（其实解法很简单）★★★★★
LeetCode 51. N-Queens：暴力解的…★★★
LeetCode 53. Maximum Subarray: 据说常见，但是一次对就好了。(Easy)
LeetCode 54. Spiral Matrix: 暴力吧。用的加访问位的方法，反正空间复杂度都是O(mn)。
LeetCode 55. Jump Game: 纯正的贪心算法，算经典题吧。
LeetCode 56. Merge Intervals: 区间[a1, b1]和[a2, b2]如果相交 -> 其与[a1, b2]和[a2, b1]的交集并无差别。如果不相交，排序的对应关系不会改变。
LeetCode 58. Length of Last Word: 会粗心的一题，适合用split()做尝试。但自己直接写算法不调用方法会更快~ ★★
LeetCode 59. Spiral Matrix II: 抽丝剥茧的做吧。
LeetCode 60. Permutation Sequence: 有关全排列。★★★★★
LeetCode 62. Unique Paths: 求解方块路径个数，组合数 -> 感觉深究还是有很多问题
LeetCode 65. Valid Number: 求String是否是正确的数字型。（这里没考虑十六进制喔）。忘记考虑的情况：”4e+”，”.”，” -.”。有些特别的方法，比如正则表达式的匹配，以及Double.parseDouble()。
LeetCode 63. Unique Paths II: 求解方块路径个数，有限制 -> 有时候存储空间小可以节省丢时间
LeetCode 64. Minimum Path Sum: 求解路径和最小，同L63.
LeetCode 66. Plus One: 数组表示的int类型的value加一
LeetCode 67. Add Binary: 二进制加法，char型和int型转换
LeetCode 69. Sqrt(x): 二分查找。Integer.MAX_VALUE求平方的结果约为46340多一点，不是0xFFFF啊…
LeetCode 70. Climbing Stairs： 基础的动态规划，有点像Fibonacci.
LeetCode 71. Simplify Path : 栈
LeetCode 73. Set Matrix Zeroes: O(m + n) 空间消耗，这样是可以的。如果是用常量空间，就会想到需要复用它本来拥有的空间，这样会产生的问题是赋值的先后顺序产生的结果会有很大的不同。常量空间值得一试。★★★★★★
LeetCode 77. Combinations : backtracing，考虑空链表的时候第一次添加元素的情况。
LeetCode 91. Decode Ways : 问有多少种可能的编码方式。需要检查是否符合编码条件(与Roman和十进制转换的区分)。
LeetCode 93. Restore IP Addresses: 递归即可（Easy），注意点: 不符合数字要求的(? 不知道是否必须)，合法的IP每节取值直接0-255之间就好。‘0’需要前边有值。或者直接为‘0’。
LeetCode 94. Binary Tree Inorder Traversal: 中序遍历。递归迭代都需要掌握。
LeetCode 98. Validate Binary Search Tree : 判断是否是BST，忽略了的是结点大小是需要整棵树来判断的，不单单是父子结点值的比较。此时用了Integer型表示是否被赋值（因为用Integer.MAX_VALUE和Integer.MIN_VALUE遇到相等的情况很难搞），也可以用long型来替换。
LeetCode 100. Same Tree : 判断两棵树是否相等。(Easy)
LeetCode 101. Symmetric Tree : 判断树是否是轴对称。适合练习Java的集合类写法。
LeetCode 102. Binary Tree Level Order Traversal: 树的层级遍历。适合练习广度优先搜索。★★★
LeetCode 104. Maximum Depth of Binary Tree : 求树的高度。直接递归比较快。(Easy)
LeetCode 105. Construct Binary Tree from Preorder and Inorder Traversal : 不难，略繁琐，直接用的递归。
LeetCode 106. Construct Binary Tree from Inorder and Postorder Traversal : 直接用的递归，可以试着用迭代。
LeetCode 107. Binary Tree Level Order Traversal II : 基本题。
LeetCode 108. Convert Sorted Array to Binary Search Tree : 直接递归做很简单。
LeetCode 110. Balanced Binary Tree: 判断是否是平衡二叉树。(Easy)
LeetCode 112. Path Sum: 叶子叶子叶子！！
LeetCode 113. Path Sum II: 别嵌套不该嵌套的函数啊。
LeetCode 114. Flatten Binary Tree to Linked List: 用了Morris Traversal基本的方法。
LeetCode 116. Populating Next Right Pointers in Each Node: 树的题比较值得做的。但感觉我用了117的方法呢…★★★★★
LeetCode 117. Populating Next Right Pointers in Each Node II: 树的题。记得考虑先后连接顺序和null的情况。然而感觉还是做麻烦了呢。★★★★★
LeetCode 120. Triangle : 动态规划，三角形中的最小值路径。之前的值可做保存，略像LeetCode 63。
LeetCode 121. Best Time to Buy and Sell Stock : 一题不晓得如何用到DP的题 (Easy)
LeetCode 122. Best Time to Buy and Sell Stock II : 有些情况会考虑不到，需要多列举几种情况。But，要是改变思考的方式，解法很简单！！！★★★
LeetCode 124. Binary Tree Maximum Path Sum :不像hard题，反而应该归为medium题才对。注意是否需要从叶节点开始。
LeetCode 125. Valid Palindrome : 在打算直接码代码提交时遇到很多小错误，就像复制粘贴以后括号不对，数字忘记0的存在，以及不审题。
LeetCode 128. Longest Consecutive Sequence: Hash table 经典题，O(N)复杂度有部分就是用hashtable的题。思路很明确。★★★★★★
LeetCode 129. Sum Root to Leaf Numbers: 没如何算，暴力解决了。可以用BFS。
LeetCode 136. Single Number : 异或运算
LeetCode 137. Single Number II : 二进制数值位
LeetCode 138. Copy List with Random Pointer : 深度拷贝链表。★★★★★
LeetCode 144. Binary Tree Preorder Traversal: LinkedList的addAll()和想象中的不一样，还是ArrayList比较好，直接System.arraycopy()了
LeetCode 145. Binary Tree Postorder Traversal: 用递归的话，难度只是Easy。用迭代的话，不改结构啊尽量。插入值时候特殊处理。
LeetCode 150. Evaluate Reverse Polish Notation: 练手题。
LeetCode 151. Reverse Words in a String: 一条看上去简单的题总是AC不过，也是烦。用了原有的方法调用，或者自己写的方法。需要考虑很多种空格情况。split()的正则写的不太好 ★★★★★
LeetCode 152. Maximum Product Subarray: 求最大子串积，我以0为界分段求了[需要着重考虑0可能是最大的情况以及被0分成只有一个元素的情况]。别人用最大子串和的求法也可以，考虑的条件会少一些。★★★
LeetCode 160. Intersection of Two Linked Lists : 链表交集
LeetCode 166. Fraction to Recurring Decimal : 这类题主要是符号与overflow的情况，overflow的情况要考虑全面，用long型代替int型扩展更简易。
LeetCode 167. Two Sum II - Input array is sorted: 和第一题相比这题更容易好不好。(Easy)
LeetCode 168. Excel Sheet Column Title: Excel表头数字转换，int型转char型直接强转就可以了，记得测试边界用例。
LeetCode 171. Excel Sheet Column Number: 与LeetCode 168相反，比它容易。
LeetCode 172. Factorial Trailing Zeroes: 值得一做。之前想考虑2和5的情况，不需要这么复杂，两者有包含关系的。
LeetCode 173. Binary Search Tree Iterator: 实现BST的iterator。实现迭代器主要是先获得值，再移动到下个位置，最后返回当时获得的值。
LeetCode 175. Combine Two Tables: SQL，left join
LeetCode 187. Repeated DNA Sequences: 暴力可解，把String编码也可解。记得编码编码编码的方法。
LeetCode 190. Reverse Bits: 二进制反转。（Easy）-> 别人的方法，分段，做成byte的反转，记录byte反转结果（memoized）。
LeetCode 198. House Robber: 动态规划，不能连着偷盗两所屋子。
LeetCode 199. Binary Tree Right Side View: 层序遍历的水题。(Easy)
LeetCode 202. Happy Number: 也可以通过链表中是否有环来判断，注意判断条件
LeetCode 204. Count Primes: 计算小于n的素数数量。经典题，Hints的内容很有意思，值得一做。
LeetCode 205. Isomorphic Strings: 很久以前审题错误了。主要问题在于，String包含的东西是什么。char型16位，最大为65536。用桶或者Map都行，前者更快，后者也许内存会小些吧。
LeetCode 207. Course Schedule: 拓扑排序经典题。★★★★★
LeetCode 209. Minimum Size Subarray Sum: 还是先判断下一步是否符合条件会比较好想。O(nlogn)的时间复杂度需要空间复杂度为O(n)，二分查找。
LeetCode 215. Kth Largest Element in an Array：做法简洁而不优雅hhh，// TODO: 用分治再做次？
LeetCode 217. Contains Duplicate : 计算是否有重复整数。(Easy)
LeetCode 219. Contains Duplicate II: 直接hash table吧，那样比较容易。
LeetCode 223. Rectangle Area : 看清要求什么。同时注意题目已给的约束。
LeetCode 224. Basic Calculator : 加减带括号的一题，不知道为什么乘除带括号就是String类的题了。标明是hard，但做起来不难。不够简便还需要化简的一题。
LeetCode 226. Invert Binary Tree: 二叉树的翻转。(Easy)
LeetCode 227. Basic Calculator II: 不带括号的加减乘除运算模拟。居然把加减法变成可从右到左计算了也是醉。★★★★★
LeetCode 228. Summary Ranges: 可用二分查找
LeetCode 230. Kth Smallest Element in a BST: Morris Traversal解决的。主要bug在于该结点还是上一个结点是我们所需要的。
LeetCode 231. Power of Two : 判断是否是2的幂，0不是2的幂，需要单独判断。
LeetCode 235. Lowest Common Ancestor of a Binary Search Tree: 递归、迭代都尝试了，居然第一个反应不是迭代也醉了。
LeetCode 238. Product of Array Except Self: 特殊问题，将问题简单化
LeetCode 242. Valid Anagram: 经典的hash table的题目。
LeetCode 257. Binary Tree Paths : 区分叶子节点和度为1的结点。当根节点特殊处理时记得处理。
LeetCode 258. Add Digits: Wiki的方法很独到 // TODO: Digital root
LeetCode 260. Single Number III: 方法巧妙，用set做可以过，但是会本身的方法也很赞。

LeetCode 264. Ugly Number II: 方法略蠢，把乘以2，乘以3，乘以5的数放了个优先队列，取队列最小值。// TODO:
LeetCode 268. Missing Number : 异或运算/不变量

LeetCode 279. Perfect Squares: 用自底向上的动态规划法搞定了。（自顶向下应该也可以）// TODO: 以后可以看看Lagrange’s Four Square theorem的方法求解。

LeetCOde 292. Nim Game : (Easy) 扩展很难，LeetCode没什么扩展。
LeetCode 297. Serialize and Deserialize Binary Tree : 由String生成二叉树，二叉树再变为String。一直都很想写的一题，居然真让实现。不难，但是估计DFS比较好。
LeetCode 299. Bulls and Cows : 这是个不要太擅长喔的游戏。
LeetCode 300. Longest Increasing Subsequence : 最长增量子串。DP方法实现简单。然而有个更简单的方法。 ★★★★★
LeetCode 303. Range Sum Query - Immutable : 求指定的数组子集的和。自己想的太复杂。可做。
LeetCode 319. Bulb Switcher : 智力题，做过了就忘不掉了的。
LeetCode 322. Coin Change: DP，看问题规模，再写循环。可想多种迭代关系式。

LeetCode 342. Power of Four : 判断是否是4的幂，和判断是否是2的幂方法并不一样。[见Solution可以直接暴力解 -> 0x55555555 ]
LeetCode 343. Integer Break : 可用动态规划。也可用数学方法做，其后的数学思想还不错。把1-4的情况特殊处理出来会简单些。
LeetCode 344. Reverse String : 直接调用Java库方法会比较容易。但自己写char型数组转换会比较速度。
LeetCode 345. Reverse Vowels of a String: 元音字母翻转。遇到String处理char型数组的，记得大小写。做存在某个char的字典，用String存储比Set存储速度，String存储，indexOf(i) == -1 表示不存在。
LeetCode 347. Top K Frequent Elements : 实现了Comparable接口做的。方法真不简便。可以用桶排序(先统计数量再桶排序)、优先队列[这个比较绝]、TreeMap(先统计数量，再用这个Map的映射反转)
LeetCode 349. Intersection of Two Arrays : 数组交集
LeetCode 350. Intersection of Two Arrays II : 数组交集
LeetCode 357. Count Numbers with Unique Digits : 排列组合
LeetCode 367. Valid Perfect Square: 同69题，二分查找。
LeetCode 368. Largest Divisible Subset: 正儿八经的动态规划。参数为数组，需要检查数组长度。
LeetCode 374. Guess Number Higher or Lower: 猜数字，二分，注意大小溢出。
LeetCode 383. Ransom Note: 练手水题。(Easy)
LeetCode 387. First Unique Character in a String: 简单hash题。(Easy)
LeetCode 389. Find the Difference: 技巧性比较强。
LeetCode 392. Is Subsequence : 倒也不像贪心算法、动态规划一类，更偏向two pointers 的类型，注意String类中charAt()和indexOf()时间复杂度的比较。
LeetCode 394. Decode String: 转换，用的递归。记录值的时候用了引用变量做值传递。用栈做，注意需要几个栈记录当前状态。
LeetCode 395. Longest Substring with At Least K Repeating Characters: 扫描出现次数小于k的字母，然后将它们作为分隔符，然后就将这些子串划分成原问题的子问题，递归求解。★★★★★★

LeetCode 400. Nth Digit: 挺繁琐的一题，值得一做。注意乘法overflow。
LeetCode 402. Remove K Digits: 贪心算法。正常难度，需要考虑的问题较多。
LeetCode 404. Sum of Left Leaves: 做这类题重要的是审题。递归比迭代快。
LeetCode 406. Queue Reconstruction by Height : 贪心算法，正常难度的题
LeetCode 409. Longest Palindrome: 超简单的一题想得太复杂了。

Write a SQL query to get the second highest salary from the Employee table.
1) select IFNULL((select distinct Salary from Employee order by Salary desc limit 1 offset 1), NULL) as SecondHighestSalary;
2) select max(Salary) as SecondHighestSalary from Employee where Salary not in (select max(Salary) from Employee)
